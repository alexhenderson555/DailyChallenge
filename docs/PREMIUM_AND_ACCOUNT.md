# Premium и привязка к аккаунту

## Как сейчас устроено

- **Покупка Premium:** нет реальной оплаты. Пользователь может:
  1. **Промокод** — ввести код в окне Premium → если код в списке действующих и ещё не использован на этом устройстве, выставляется Premium.
  2. **Тестовая кнопка** — «Получить Premium» выставляет флаг без оплаты (для отладки).

- **Хранение:** флаг `isPremium` и список использованных промокодов хранятся **только локально** (DataStore на устройстве). При переустановке или смене устройства Premium не восстанавливается.

- **Промокоды:** действующие коды задаются в `PromoCodes.kt`. Один и тот же код нельзя активировать дважды на одном устройстве (использованные коды сохраняются локально).

---

## Как сделать привязку к аккаунту (на другом устройстве тоже)

Чтобы Premium был у пользователя на любом устройстве после входа в аккаунт:

1. **Авторизация**  
   Добавить вход по аккаунту (Google, email, телефон и т.п.), например через Firebase Auth. В приложении хранить `userId` (или токен).

2. **Backend**  
   Сервер хранит для каждого пользователя:
   - `user_id` (из авторизации);
   - `is_premium: bool`;
   - при необходимости: таблица промокодов и кто какой код уже использовал.

3. **При запуске приложения**  
   Если пользователь авторизован: запрос к backend «дать статус Premium для этого user_id». Ответ `is_premium` записать в локальный DataStore (вызвать `setPremium(...)`), чтобы весь текущий код продолжал работать.

4. **Активация промокода**  
   Вместо локальной проверки из `PromoCodes.kt`: отправить на backend запрос «активировать промокод для user_id». Сервер:
   - проверяет код;
   - помечает код как использованный этим пользователем (или вообще один раз на весь код);
   - выставляет пользователю `is_premium = true`;
   - возвращает успех/ошибку.  
   Приложение при успехе вызывает `setPremium(true)` и при необходимости обновляет UI.

5. **Новое устройство**  
   Пользователь входит в тот же аккаунт → приложение снова запрашивает у backend статус Premium по `user_id` → получает `is_premium: true` → вызывает `setPremium(true)` локально. Premium доступен и на этом устройстве.

Итого: текущая логика (DataStore, `setPremium`, экран Premium, промокоды) остаётся; добавляются авторизация, запрос статуса при старте и проверка промокода через backend.
